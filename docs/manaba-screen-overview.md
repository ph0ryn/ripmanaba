# manaba画面情報メモ

## 目的

manabaの各画面にどのような情報が表示されているかを整理する。

このメモではDOM構造、selector、HTTP requestの詳細は扱わない。それらは後続
のスクレイピング方法調査で別途確認する。

一方で、ページ遷移に使うpathはCLI設計に必要なので、このメモにも記録する。

## URLの扱い

manabaのURLは大学や組織ごとにdomainが異なる。

そのため、ripmanabaではmanabaのdomainをハードコードしない。`ripmanaba
auth`でユーザーが開いたmanabaページを起点に、認証済みのoriginを記録して
後続コマンドで使う。

pathはmanaba側で共通とみなして調査する。実装では、記録したoriginと共通
pathを組み合わせて画面へ遷移する。

確認したURL構造:

```text
https://<manaba-domain>/ct/<path>
```

確認したpath:

| 画面 | path |
| --- | --- |
| ホーム | `/ct/home` |
| コース一覧 | `/ct/home_course` |
| 未提出課題一覧 | `/ct/home_library_query` |
| 提出記録 | `/ct/home_submitlog` |
| ポートフォリオ | `/ct/home_coursetable` |
| コース詳細 | `/ct/course_<course-id>` |
| レポート一覧 | `/ct/course_<course-id>_report` |
| レポート詳細 | `/ct/course_<course-id>_report_<report-id>` |
| 小テスト一覧 | `/ct/course_<course-id>_query` |
| 小テスト詳細 | `/ct/course_<course-id>_query_<query-id>` |

## 共通で表示される情報

ログイン後の各画面には、おおむね以下の情報が表示される。

- 現在ログインしているユーザーの表示領域
- 言語切替
- 設定
- ログアウト
- マイページ
- コース
- メモ一覧
- マニュアル

コース内の画面では、これに加えてコース名、担当教員、年度、開講時期、曜日
時限などのコース概要が表示される。

## 意味的な構造

初期実装で扱う意味的なresourceは次のように整理する。

```text
manaba
├── 全体お知らせ
│   └── お知らせ
├── コース
│   ├── コースニュース
│   ├── 小テスト
│   ├── アンケート
│   ├── レポート
│   ├── 成績
│   ├── コースコンテンツ
│   └── 提出記録
├── 未提出課題
│   └── 課題[]
│       ├── 小テスト
│       ├── アンケート
│       ├── レポート
│       └── プロジェクト
└── 提出記録
    └── 提出[]
        ├── 小テスト
        ├── アンケート
        ├── ドリル
        ├── レポート
        └── プロジェクト
```

未提出課題と提出記録は、それぞれ複数コースを横断する配列として扱う。

後回しにするresource:

```text
コース
├── シラバス
├── プロジェクト
├── 掲示板
│   └── スレッド
├── グループニュース
├── 個別指導
└── ピアレビュー
```

初期実装では扱わないresource:

```text
ユーザー
└── ポートフォリオ
```

## ホーム画面

ホーム画面はログイン後のダッシュボードとして機能している。

表示される情報:

- お知らせ
- コース一覧
- 課題
- 未提出課題
- 提出記録
- ポートフォリオへの導線
- 表示形式の切替

コース一覧は複数の表示形式を持つ。

- サムネイル
- リスト
- 曜日

曜日表示では、曜日と時限ごとにコースが配置される。

## お知らせ

ホーム画面には組織からのお知らせが表示される。

表示される情報:

- 日付
- お知らせタイトル
- 詳細へのリンク

現時点ではCLIの主要対象ではないが、ホーム画面の情報として存在する。

追加コマンド候補:

- `ripmanaba notice ls`
- `ripmanaba notice info <notice-id>`
- `ripmanaba notice open <notice-id>`

## コース一覧画面

コース一覧画面では、履修中または参照可能なコースが表示される。

表示される情報:

- コース名
- 年度
- 開講情報
- 担当教員
- コース詳細へのリンク
- お気に入り状態またはお気に入り操作

ホーム画面と同様に、サムネイル、リスト、曜日の表示形式がある。

`ripmanaba course ls`では、まずこの画面の情報が候補になる。

## コース詳細画面

コース詳細画面では、対象コースの情報とコース内メニューが表示される。

表示される情報:

- コース名
- 担当教員
- 年度
- 開講時期
- 曜日・時限
- コースニュース
- スレッド
- コンテンツ
- 各機能へのメニュー

確認できたコース内メニュー:

- シラバス
- コースニュース
- 小テスト
- アンケート
- レポート
- プロジェクト
- 成績
- 掲示板
- コースコンテンツ
- 提出記録
- グループニュース
- 個別指導
- ピアレビュー

`ripmanaba course info [course ID]`では、この画面のコース概要とメニュー
状態が候補になる。

コースコンテンツは、深いコマンド階層にはせず `content` resourceとして扱う
候補にする。

追加コマンド候補:

- `ripmanaba content ls [course-id]`
- `ripmanaba content info <content-id>`
- `ripmanaba content open <content-id>`

## 未提出課題一覧画面

未提出課題一覧画面では、未提出または未完了の課題が一覧表示される。

表示される情報:

- タイプ
- タイトル
- コース
- 受付開始日時
- 受付終了日時
- 受付期間

タイプには、少なくとも以下が表示される。

- レポート
- 小テスト

各行には、課題タイプ、課題タイトル、コースへのリンクがある。

`ripmanaba task ls`では、まずこの画面の情報が候補になる。

## 課題詳細画面

課題タイトルから詳細画面へ遷移できる。

レポート課題の詳細画面で表示される情報:

- 課題タイトル
- 問題文
- 受付開始日時
- 受付終了日時
- ポートフォリオまたは閲覧設定
- 学生による再提出の許可
- 添付ファイル
- 提出状態
- アップロード操作
- 課題一覧へ戻る導線

`ripmanaba task info [task ID]`では、この画面の情報が候補になる。

## 提出記録画面

提出記録画面では、過去の提出履歴を確認できる。

表示される情報:

- 対象期間
- 絞り込み
- 種別フィルタ
- 期間フィルタ
- ページ送り

確認できた種別フィルタ:

- すべて
- 小テスト
- アンケート
- ドリル
- レポート
- プロジェクト

確認できた期間フィルタ:

- 7日
- 30日
- 90日
- 180日

READMEの最低要件には入っていないが、提出済み課題や履歴系コマンドの候補に
なる。

追加コマンド候補:

- `ripmanaba submission ls`
- `ripmanaba submission info <submission-id>`
- `ripmanaba submission open <submission-id>`

## ポートフォリオ画面

ポートフォリオ画面では、年度ごとのコースや提出物に関連する情報が表示され
る。

表示される情報:

- ユーザーのポートフォリオ
- プロフィール
- オープンバッジ設定
- 年度ごとの区分
- コースごとの項目
- 提出物や成績に関連する導線

READMEの最低要件には入っていないが、将来的な対象候補として画面が存在する。

## 次に調べること

次の調査では、画面上の情報をどのように安定して取得できるかを確認する。

- 各画面のDOM構造
- 安定して使えるselector
- 課題タイプごとの詳細画面差分
- コース一覧の表示形式ごとの差分
- 直接HTTPで取得できる画面とPlaywrightが必要な画面の切り分け
- `open`系コマンドで開くべきURLの扱い
